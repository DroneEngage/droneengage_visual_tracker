# Tracker Module Messages Documentation

## Overview

This document details all DroneEngage protocol messages handled and generated by the Tracker module. The tracker processes both direct tracking commands and YOLO AI recognition messages for coordinated object tracking functionality.

## Message Dependencies

### YOLO AI Integration
The tracker module depends on YOLO AI modules for object detection and tracking:

**Message Filter Configuration** (from `main.cpp`):
```cpp
#define MESSAGE_FILTER {TYPE_AndruavMessage_TrackingTarget_ACTION,\
                        TYPE_AndruavMessage_TrackingTargetLocation,\
                        TYPE_AndruavMessage_TargetTracking_STATUS,\
                        TYPE_AndruavMessage_AI_Recognition_STATUS,\
                        TYPE_AndruavMessage_AI_Recognition_TargetLocation,\
                        TYPE_AndruavMessage_CONFIG_ACTION, \
                        TYPE_AndruavMessage_DUMMY}
```

**Dependency Flow**:
```
YOLO AI Module → Tracker Module → GCS/Controller
     ↓                    ↓              ↓
AI Detection → Track Processing → Status Updates
```

## Input Messages (Handled by Parser)

### 1. TYPE_AndruavMessage_TrackingTarget_ACTION (1042)
**Purpose**: Direct tracking control commands

**Message Structure**:
```json
{
  "mt": 1042,
  "ms": {
    "a": <action_id>,
    "b": <float>,  // X coordinate (POINT/REGION)
    "c": <float>,  // Y coordinate (POINT/REGION)  
    "d": <float>,  // Width (REGION only)
    "e": <float>,  // Height (REGION only)
    "r": <float>   // Radius (POINT only)
  }
}
```

**Supported Actions**:

| Action ID | Action Name | Description | Parameters |
|-----------|-------------|-------------|------------|
| 0 | `TrackingTarget_ACTION_TRACKING_POINT` | Track circular region | `b,c,r`: X,Y,Radius |
| 1 | `TrackingTarget_ACTION_TRACKING_REGION` | Track rectangular region | `b,c,d,e`: X,Y,Width,Height |
| 2 | `TrackingTarget_ACTION_TRACKING_STOP` | Stop tracking | None |
| 3 | `TrackingTarget_ACTION_TRACKING_PAUSE` | Pause tracking | None |
| 4 | `TrackingTarget_ACTION_TRACKING_ENABLE` | Enable tracking | None |
| 5 | `TrackingTarget_ACTION_TRACKING_QUERY_CONFIG` | Request configuration | None |
| 6 | `TrackingTarget_ACTION_TRACKING_AI_DRIVER_ENABLED` | Enable AI priority mode | None |
| 7 | `TrackingTarget_ACTION_TRACKING_AI_DRIVER_DISABLED` | Disable AI priority mode | None |

**Parser Location**: `tracker_andruav_message_parser.cpp` - `parseCommand()` method

### 2. TYPE_AndruavMessage_AI_Recognition_STATUS (1077) ⚠️ YOLO Dependency
**Purpose**: Receive AI recognition status updates from YOLO modules

**Message Structure**:
```json
{
  "mt": 1077,
  "ms": {
    "a": <status_code>
  }
}
```

**Status Codes**:
- 0: `TrackingTarget_STATUS_AI_Recognition_LOST`
- 1: `TrackingTarget_STATUS_AI_Recognition_DETECTED`
- 2: `TrackingTarget_STATUS_AI_Recognition_ENABLED`
- 3: `TrackingTarget_STATUS_AI_Recognition_DISABLED`

**Processing**: Updates tracker's AI status and adjusts tracking behavior accordingly. 
**AI Feedback Integration**: Uses `m_tracker_main.setAITrackerStatus(status)`  to synchronize AI recognition status with the tracking engine.

### 3. TYPE_AndruavMessage_AI_Recognition_TargetLocation (1078) ⚠️ YOLO Dependency
**Purpose**: Receive detected object locations from YOLO modules

**Message Structure**:
```json
{
  "mt": 1078,
  "ms": {
    "t": [<targets_array>]  // All detected objects
  }
}
```

OR

```json
{
  "mt": 1078,
  "ms": {
    "b": {                // Best/targeted object
      "x": <float>,      // X coordinate
      "y": <float>,      // Y coordinate  
      "w": <float>,      // Width
      "h": <float>       // Height
    }
  }
}
```

**Processing**: 
- "t" field: Logs all detected objects for debugging
- "b" field: Extracts best object coordinates and forwards to tracking engine

**AI Feedback Integration**:
The tracker module uses AI feedback to update tracking coordinates in real-time:
1. **Status Updates**: `m_tracker_main.setAITrackerStatus(status)` synchronizes AI recognition status with the tracker
2. **Location Updates**: `m_tracker_main.onAITrackerBestRect(x, y, w, h)` forwards the best detected object coordinates to the tracking engine
3. **Coordinate Processing**: Converts YOLO bounding box coordinates to tracker coordinate system (center point calculation)

### 4. TYPE_AndruavMessage_RemoteExecute (1005)
**Purpose**: Execute remote commands on the tracker module

**Message Structure**:
```json
{
  "mt": 1005,
  "ms": {
    "C": <command_id>
  }
}
```

**Current Implementation**: Debug logging only (placeholder for future expansion)

## Output Messages (Generated by Facade)

### 1. TYPE_AndruavMessage_TrackingTargetLocation (1043)
**Purpose**: Send current tracking target location

**Message Structure**:
```json
{
  "mt": 1043,
  "ms": {
    "t": <targets_data>  // Current tracking targets
  }
}
```

**Generated By**: `sendTrackingTargetsLocation()` - Sends current tracked object locations

**Facade Location**: `tracker_facade.cpp`

### 2. TYPE_AndruavMessage_TargetTracking_STATUS (1044)
**Purpose**: Send tracker status information

**Message Structure**:
```json
{
  "mt": 1044,
  "ms": {
    "a": <status_code>,
    "b": <camera_direction>
  }
}
```

**Status Codes**:
- Various tracking states (implementation-specific)
- Camera direction indicates tracking orientation

**Generated By**:
- `sendTrackingTargetStatus()` - Sends current tracking status
- `sendTrackingConfig()` - Sends configuration response

**Facade Location**: `tracker_facade.cpp`

## Message Flow Diagrams

### Direct Tracking Control Flow
```
GCS/Controller → Tracker Module
├── TrackingTarget_ACTION (1042) - POINT with coordinates
│   └── Response: TrackingTargetLocation (1043) with tracked position
├── TrackingTarget_ACTION (1042) - REGION with bounds
│   └── Response: TrackingTargetLocation (1043) with tracked region
├── TrackingTarget_ACTION (1042) - STOP
│   └── Response: TargetTracking_STATUS (1044) with stopped status
└── TrackingTarget_ACTION (1042) - QUERY_CONFIG
    └── Response: TargetTracking_STATUS (1044) with config data
```

### YOLO AI Integration Flow
```
YOLO AI Module → Tracker Module → GCS/Controller
├── AI_Recognition_TargetLocation (1078) - "b" field: Best object
│   └── Processing: Extract coordinates, update tracking
├── AI_Recognition_TargetLocation (1078) - "t" field: All objects
│   └── Processing: Debug logging only
└── AI_Recognition_STATUS (1077) - Status updates
    └── Processing: Update AI tracker status
```

### Combined Tracking Flow
```
YOLO AI → Tracker → GCS
Detection → Processing → Status Updates
    ↓         ↓           ↓
 Objects → Coordinates → Location Reports
```

## YOLO AI Integration Details

### Message Processing
1. **Receives AI Detection**: Processes `AI_Recognition_TargetLocation` messages
2. **Extracts Coordinates**: Converts YOLO bounding boxes to tracking coordinates
3. **Updates Tracking**: Forwards processed coordinates to tracking engine
4. **Status Synchronization**: Maintains AI status for coordinated behavior

### Coordinate Transformation
```cpp
// From YOLO AI message
const float x = obj["x"].get<float>();
const float y = obj["y"].get<float>();
const float w = obj["w"].get<float>();
const float h = obj["h"].get<float>();

// To tracker coordinate system
// values ranges from -0.5 to + 0.5
const float centerX = -0.5 + x + w / 2.0;
const float centerY = -0.5 + y + h / 2.0;
```

### Debug Integration
- Logs all AI objects for comprehensive debugging
- Highlights best object selection and coordinate extraction
- Provides visibility into AI-tracker interaction
